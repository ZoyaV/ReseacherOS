Проект уже на ResearchOS — только запустить

Дерево гипотез уже есть у команды (ветка koi/research). Нужно поставить движок, повесить research-ветку в tree/ и открыть UI.

Как это устроено

workspace/
├── ReseachOS/                         ← движок
├── tree/
│   ├── TalkingHeads/koi-structure/    ← ветка koi/research
│   └── verl-agent-craftext/koi-structure/
├── TalkingHeads/                      ← код, любая ветка
└── verl-agent-craftext/

Сервер смотрит соседние папки: если встречается каталог tree, на следующем уровне ищет */koi-structure/project.md.

Пошагово

  1. Клон движка 
    git clone git@github.com:ZoyaV/ReseacherOS.git ReseachOS
cd ReseachOS
./scripts/koi-serve.sh start
  2. Код-репо рядом с ReseachOS 
    cd ..
git clone <url-вашего-code-репо> TalkingHeads
  3. Подтянуть koi/research в tree/ 
    cd ReseachOS
python -m koi.projects.install_cli install TalkingHeads
# статус всех соседей:
python -m koi.projects.install_cli status

    Если ветка koi/research уже на remote — появится tree/TalkingHeads/ (git worktree). Если материалы лежали по-старому внутри code-репо — CLI смигрирует их в tree/.

  4. Запуск 
    ./scripts/koi-serve.sh restart
# интерфейс  http://127.0.0.1:8080
# API        http://127.0.0.1:8010/docs

Дальше синхронизация исследования: кнопка Sync в UI или python -m koi.projects.sync_cli pull|push --project-id <id>. Если ветки ещё нет — путь «впервые подключаю свой код».
