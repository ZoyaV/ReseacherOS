Впервые подключаю свой код

Код эксперимента уже есть. Материалы исследования живут отдельно: tree/<репо>/koi-structure/ на ветке koi/research, а код остаётся в <репо>/ на любой ветке.

Целевая раскладка

workspace/
├── ReseachOS/                      ← движок
├── tree/
│   └── my_experiment/
│       └── koi-structure/          ← ветка koi/research
└── my_experiment/                  ← код, любая ветка
    └── …
  1. Поставьте ResearchOS рядом с проектом 
    git clone git@github.com:ZoyaV/ReseacherOS.git ReseachOS
# рядом уже лежит (или клонируете) my_experiment/
  2. Установите проект одной командой 
    cd ReseachOS
./scripts/koi-serve.sh start   # поднимет .venv при необходимости
python -m koi.projects.install_cli install my_experiment
# или: ./scripts/koi-project-install.sh install my_experiment

    CLI сам определит кейс: создать orphan koi/research, повесить worktree в tree/, или смигрировать старый my_experiment/koi-structure.

  3. (Опционально) Дерево гипотез с AI-помощником

    Если материалов ещё нет — после install попросите онбординг:

    Подключи ../my_experiment к ResearchOS.
Следуй agents/skills/koi-project-onboard/SKILL.md.
Пиши в tree/my_experiment/koi-structure/.
  4. Откройте интерфейс

    ./scripts/koi-serve.sh restart → http://127.0.0.1:8080. Проект появится в сайдбаре.

Зачем tree/ и отдельная ветка

Код и карта исследования — разные ритмы работы. Ветка-сирота koi/research хранит только koi-structure/; worktree в tree/<репо>/ держит её открытой рядом с движком, не смешивая с feature-ветками кода.

Посмотреть статус всех соседей: 

python -m koi.projects.install_cli status
