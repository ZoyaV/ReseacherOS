# Сравнить текст и картинку на телефоне внутри шести категорий таргетинга

Статус: запланирован  
Канбан: fmt-strat-targeting  
Зависимости: fmt-aggregate, fmt-paired → эта карточка → fmt-strat-weak-targeting

---

## 1. Описание эксперимента и зачем

Сравнение внутри рекламных групп (fmt-paired) не отделяет эффект формата от типа поискового запроса: одна группа может собирать и целевые, и широкие запросы. В отчёте Директа есть колонка «Категория таргетинга» — шесть типов запросов. Нужно проверить, сохраняется ли провал картинки на телефоне, если сравнивать форматы внутри одной категории intent.

Цель: для каждой категории таргетинга на мобильных устройствах сравнить долю кликов (CTR) текстового и графического формата; там, где оба формата имеют достаточный объём, зафиксировать разрыв в процентных пунктах.

Зачем в гипотезе: если текст стабильно обгоняет картинку внутри «Целевых запросов» (а не только в агрегате по кампании), слабость графики на телефоне вряд ли объясняется только смешением разных типов запросов между группами.

Что не является целью:

- повторять сводку fmt-aggregate или парное сравнение по 25 группам;
- сравнивать категории между собой по CPC (это карточка seg-targeting);
- рекомендации по отключению графики (блок r-format-policy).

---

## 2. Основная метрика (финальный тест)

В каждой категории таргетинга, где на телефоне крутятся оба формата и в каждой ячейке не меньше 100 показов, — разница CTR текст минус CTR картинка. Успех: в каждой такой категории текст даёт более высокую долю кликов; ожидается минимум одна категория с разрывом ≥ 3 п.п. (ориентир — «Целевые запросы»).

### Определение

| Параметр | Значение |
|----------|----------|
| Метрика | доля кликов (CTR) отдельно для текстового и графического формата на телефоне |
| Единица наблюдения | одна категория таргетинга |
| Фильтр | «Тип устройства» = мобильные; в каждой ячейке «категория × формат» ≥ 100 показов |
| Категории | целевые, узкие, с упоминанием конкурентов, альтернативные, широкие, не определено |
| Ориентир «Целевые запросы» | текст ~12,1% CTR (6 618 показов), картинка ~5,0% (259 показов), разрыв ~7,1 п.п. |
| Протокол | выгрузка Centra Market — велосипеды, май 2024; колонки «Категория таргетинга», «Формат», «Тип устройства», «Показы», «Клики», «Расход (руб.)» |
| Скрипт / артефакты | лок.: `datasets/bicycle.csv`; табл. A и B в §4.1 |

## 3. Постановки и подготовка

### 3.1 Агрегация «категория × формат» на мобильных → §4

Кратко: оставить строки с «Тип устройства» = мобильные; сгруппировать по «Категория таргетинга» и «Формат»; посчитать показы, клики, расход, CTR, CPC.

- Используемые данные: выгрузка Centra Market — велосипеды, май 2024; лок.: `datasets/bicycle.csv`
- Порог парного сравнения: ≥ 100 показов на каждый формат внутри категории
- Статус: запланирован → [§4](#4-эксперимент-1--сравнение-внутри-категорий)

Подзадачи:

- [ ] Заполнить табл. A: все шесть категорий × оба формата (пустые ячейки, если формат не крутился)
- [ ] Выделить в табл. B только категории, прошедшие порог 100 показов на формат
- [ ] Для каждой строки табл. B посчитать разрыв CTR (текст − картинка) в п.п.
- [ ] В §4.2 зафиксировать: в скольких категориях текст выигрывает; достаточен ли разрыв в «Целевых запросах»

#### Сбор данных

```bash
python3 -c "
import csv
from collections import defaultdict
d = defaultdict(lambda: {'i': 0, 'c': 0, 's': 0.0})
for r in csv.DictReader(open('datasets/bicycle.csv', encoding='utf-8')):
    if r['Тип устройства'] != 'мобильные':
        continue
    k = (r['Категория таргетинга'], r['Формат'])
    d[k]['i'] += float(r['Показы'] or 0)
    d[k]['c'] += float(r['Клики'] or 0)
    d[k]['s'] += float(r['Расход (руб.)'] or 0)
for k, v in sorted(d.items(), key=lambda x: (-x[1]['s'], x[0][0])):
    ctr = v['c'] / v['i'] * 100 if v['i'] else 0
    cpc = v['s'] / v['c'] if v['c'] else 0
    print(k[0], '|', k[1], f\"{v['i']:.0f} impr\", f\"CTR {ctr:.1f}%\", f\"{v['s']:.0f}₽\", f\"CPC {cpc:.1f}₽\")
"
```
