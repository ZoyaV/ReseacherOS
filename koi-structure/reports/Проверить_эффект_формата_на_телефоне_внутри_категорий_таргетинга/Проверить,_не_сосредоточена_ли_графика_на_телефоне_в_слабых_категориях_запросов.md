# Проверить, не сосредоточена ли графика на телефоне в слабых категориях запросов

Статус: запланирован  
Канбан: fmt-strat-weak-targeting  
Зависимости: fmt-strat-targeting → эта карточка → блок устранения (политика форматов)

---

## 1. Описание эксперимента и зачем

Возможное возражение к гипотезе: картинка на телефоне кажется слабой только потому, что алгоритм или настройки кампании отдают графику в «плохие» типы запросов — широкие, конкурентные, альтернативные. Нужно проверить обратное: где именно на мобильных тратится бюджет на графический формат и совпадает ли это с слабыми категориями таргетинга.

Цель: разделить категории таргетинга на «сильные» (целевые + узкие) и «слабую тройку» (широкие + конкурентные + альтернативные); для каждого блока посчитать долю расхода и показов графики на телефоне; там, где оба формата есть, сравнить CTR.

Зачем в гипотезе: если графика на телефоне почти не крутится в слабых категориях, а разрыв текст vs картинка сохраняется в сильных, провал формата — не артефакт перекоса бюджета в слабые запросы.

Что не является целью:

- полный аудит шести категорий по CPC (карточка seg-targeting);
- симуляция отключения графики (fmt-policy-off / fmt-policy-realloc);
- сравнение внутри отдельных рекламных групп (fmt-paired).

---

## 2. Основная метрика (финальный тест)

Доля расхода графического формата на телефоне в «слабой тройке» категорий vs в «сильных» категориях. Успех гипотезы подтверждается, если: (1) в слабой тройке графика на мобильных почти отсутствует или даёт ничтожную долю бюджета; (2) в сильных категориях при наличии обоих форматов текстовый CTR выше графического (ориентир: объединённые целевые + узкие — текст ~11,4%, картинка ~5,0%).

## 3. Постановки и подготовка

### 3.1 Карта расхода графики на телефоне по блокам категорий → §4

Кратко: агрегировать мобильные строки по блоку (слабая тройка / сильные / прочие) и формату; посчитать показы, клики, расход, долю бюджета графики.

- Используемые данные: выгрузка Centra Market — велосипеды, май 2024; лок.: `datasets/bicycle.csv`
- Блок «прочие»: категория «не определено» и любые прочие с ненулевым расходом
- Статус: запланирован → [§4](#4-эксперимент-1--блоки-категорий)

Подзадачи:

- [ ] Заполнить табл. A: слабая тройка vs сильные vs прочие — текст и графика отдельно
- [ ] В табл. B посчитать долю расхода графики на телефоне внутри каждого блока
- [ ] В табл. C зафиксировать CTR текст vs картинка там, где графика ненулевая
- [ ] В §4.2 ответить: сосредоточена ли графика в слабых категориях; сохраняется ли разрыв CTR в сильных

#### Сбор данных

```bash
python3 -c "
import csv
from collections import defaultdict

WEAK = {'Широкие запросы', 'Запросы с упоминанием конкурентов', 'Альтернативные запросы'}
STRONG = {'Целевые запросы', 'Узкие запросы'}

def block(cat):
    if cat in WEAK: return 'слабая тройка'
    if cat in STRONG: return 'сильные'
    return 'прочие'

d = defaultdict(lambda: {'i': 0, 'c': 0, 's': 0.0})
for r in csv.DictReader(open('datasets/bicycle.csv', encoding='utf-8')):
    if r['Тип устройства'] != 'мобильные':
        continue
    k = (block(r['Категория таргетинга']), r['Формат'])
    d[k]['i'] += float(r['Показы'] or 0)
    d[k]['c'] += float(r['Клики'] or 0)
    d[k]['s'] += float(r['Расход (руб.)'] or 0)

for k, v in sorted(d.items()):
    ctr = v['c'] / v['i'] * 100 if v['i'] else 0
    print(k[0], '|', k[1], f\"{v['s']:.0f}₽\", f\"CTR {ctr:.1f}%\")
"
```
