## 1. Описание эксперимента и зачем

Отдельно по рекламным группам и по категориям таргетинга перекос бюджета уже виден. Неясно, совпадают ли они: например, слабая отдача группы «Велосипед Каталог» может быть из‑за широких запросов внутри группы или из‑за другого состава типов запросов.

Цель: для каждой крупной рекламной группы (расход не меньше 500 ₽) посчитать долю расхода в слабой тройке типов запросов — широкие, с упоминанием конкурентов, альтернативные — и сопоставить её с долей кликов на показ (CTR) группы из отчёта по 25 группам. Успех: в §4.2 для всех групп с долей кликов на показ ниже 9% зафиксирована доля слабой тройки; названа минимум одна группа-исключение, где низкая отдача не объясняется слабыми категориями (доля слабой тройки ниже 10%); при наличии — отмечена группа с долей слабой тройки выше 10%.

Зачем в гипотезе: если перекос по группам совпадает с перекосом по типам запросов, урезать бюджет можно точечно — и по группе, и по категории. Если нет, причина слабой отдачи, вероятно, в другом — формулировка объявления, географическое намерение в запросе — а не только в типе запроса.

Что не является целью:

- повторять сводные таблицы по 25 группам или шести категориям без связки;
- симулировать перенос бюджета (отчёты по перераспределению бюджета на исторических данных);
- разбирать формат объявления и устройство (другая причина в дереве).

---

## 2. Основная метрика (финальный тест)

Доля расхода рекламной группы в слабой тройке категорий таргетинга — широкие, с упоминанием конкурентов, альтернативные — в процентах от расхода группы. В выборку входят группы с расходом не меньше 500 ₽ за период выгрузки.

### Определение

| Параметр | Значение |
|----------|----------|
| Метрика | доля расхода группы в слабой тройке, %; вторично — доля кликов на показ (CTR) и цена клика (CPC) группы |
| Слабая тройка | широкие запросы, запросы с упоминанием конкурентов, альтернативные запросы (как в отчёте по шести категориям таргетинга) |
| Порог группы | расход группы ≥ 500 ₽ (ожидается 15 групп из 25) |
| Ориентиры | «Веломагазин» ~22% расхода в слабой тройке (~1 150 ₽); «Велосипед Каталог» ~5,5% при доле кликов на показ 8,4% |
| Протокол | выгрузка Centra Market — велосипеды; двойная группировка по рекламной группе и категории таргетинга; тот же охват строк, что в отчётах по 25 группам и шести категориям (50 005 ₽, 1 391 клик) |
| Скрипт / артефакты | табл. A и B в §4.1 |

## 3. Постановки и подготовка

### 3.1 Кросс-таблица группа × категория → §4

Кратко: агрегировать показы, клики и расход по парам «рекламная группа × категория таргетинга»; для каждой группы с расходом ≥ 500 ₽ посчитать долю слабой тройки и сопоставить с долей кликов на показ (CTR) из отчёта «Распределение бюджета и доли кликов по 25 рекламным группам».

- Используемые данные: выгрузка Centra Market — велосипеды; лок.: `datasets/bicycle.csv` (все строки файла, без фильтра по «Дата» — как в двух предшественниках); табл. A отчёта по 25 группам — CTR и расход по группам
- Слабая тройка: те же три категории, что в отчёте «Сравнить шесть категорий запросов в отчёте Директа»
- Отличие от предыдущих прогонов: не отдельная агрегация по одному измерению, а связка двух — расход группы режется по категории таргетинга
- Статус: запланирован → [§4](#4-эксперимент-1--группа--категория-таргетинга)

Подзадачи:

#### Сбор данных

Протокол совпадает с отчётами по 25 группам и шести категориям: весь файл `bicycle.csv`, итоги 50 005 ₽ и 1 391 клик.

```bash
cd bicycle_problem
python3 -c "
import csv
from collections import defaultdict
WEAK={'Широкие запросы','Запросы с упоминанием конкурентов','Альтернативные запросы'}
cross=defaultdict(lambda:defaultdict(lambda:{'s':0.0,'c':0,'i':0}))
gt=defaultdict(lambda:{'s':0.0,'c':0,'i':0})
for r in csv.DictReader(open('datasets/bicycle.csv',encoding='utf-8')):
    g,t=r['Группа'],r['Категория таргетинга']
    s=float(r['Расход (руб.)'] or 0); c=float(r['Клики'] or 0); i=float(r['Показы'] or 0)
    cross[g][t]['s']+=s; cross[g][t]['c']+=c; cross[g][t]['i']+=i
    gt[g]['s']+=s; gt[g]['c']+=c; gt[g]['i']+=i
for g,v in sorted(gt.items(),key=lambda x:-x[1]['s']):
    if v['s']<500: continue
    ws=sum(cross[g][t]['s'] for t in WEAK)
    pct=ws/v['s']*100 if v['s'] else 0
    ctr=v['c']/v['i']*100 if v['i'] else 0
    print(f'{g}: weak {ws:.0f}₽ ({pct:.1f}%), group CTR {ctr:.1f}%')
"
```
