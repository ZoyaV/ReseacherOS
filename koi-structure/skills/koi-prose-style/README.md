# Тексты на понятном языке

Скилл смотрит **любой текст, который увидит человек** в проекте ResearchOS:
заголовки и описания на дереве гипотез, карточки канбана, формулировки вопросов,
рассказ о выводе, заметки в базе знаний.

Он делает две вещи:

1. Держит язык понятным для нового человека (без смеси языков, без внутренних
   идентификаторов в видимом тексте, с расшифровкой сокращений).
2. Убирает шаблонный тон языковых моделей по
   [чеклисту на Wikipedia о признаках такого письма](https://en.wikipedia.org/wiki/Wikipedia:Signs_of_AI_writing).

## Что проверяет

| Смотрит | Как лучше не писать | Как лучше |
|---------|---------------------|-----------|
| Пустые «важные» слова | революционный, многогранный, «бесшовный», «прорывной» без факта | назвать конкретный факт |
| Горлочистки | «Стоит отметить, что…», «Важно понимать…» | сразу смысл |
| Риторика | «не только X, но и Y»; всегда ровно три пункта | прямое утверждение |
| Значимость без факта | «знаменует сдвиг», «открывает горизонты» | кто выигрывает и на сколько |
| Размытые ссылки | «исследователи считают…» без имён | назвать источник или убрать |
| Заголовок на карте | длинный протокол в ярлыке | до восьми слов; детали — в описании |

Полный короткий чеклист для агента — в файле `anti-ai-writing.md`.

## Зачем он нужен

Без такой проверки текст в интерфейсе быстро становится смесью жаргона
и рекламного тона. Скилл переписывает: сначала смысл своими словами,
потом при необходимости сокращение в скобках
(например «доля кликов по показам (CTR)»).

## Как пользоваться

В установленном ResearchOS скилл включается, когда агент пишет или правит
текст для человека. Агент следует файлу `SKILL.md`; примеры — в `examples.md`.

## Состав пакета

- `SKILL.md` — правила и цикл ревью для агента
- `anti-ai-writing.md` — что проверять по чеклисту Wikipedia
- `examples.md` — примеры удачных и слабых формулировок
- `manifest.yaml` — карточка для каталога Hub
