# Подключение проекта к ResearchOS

Когда у вас уже есть код эксперимента, а дерева гипотез ещё нет, этот скилл
ведёт агента через короткий диалог: в чём проблема, какие причины вы
подозреваете, как будете проверять. В конце появляется структура проекта
и отдельная ветка для материалов исследования.

## Зачем он нужен

Без онбординга легко получить пустые узлы с внутренними id или смесь
английского и русского в одной фразе. Скилл держит разговор на естественном
языке, сверяет формулировки с «холодным читателем» и только потом пишет
файлы проекта.

## Как пользоваться

В установленном ResearchOS попросите ассистента подключить репозиторий
или «сделать онбординг». Агент следует `SKILL.md` в
`agents/skills/koi-project-onboard/` (есть зеркало в `.cursor/skills/`).

## Состав пакета

- `SKILL.md` — сценарий для агента
- `framing-checks.md` — проверки формулировок
- `onboard-brief.template.md` — шаблон брифа
- `manifest.yaml` — карточка для каталога Hub
