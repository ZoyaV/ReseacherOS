# Подключение проекта к ResearchOS

Когда у вас уже есть код эксперимента, а дерева гипотез ещё нет, этот скилл
ведёт агента через короткий диалог: в чём проблема, какие причины вы
подозреваете, как будете проверять. В конце появляется структура проекта
и отдельная ветка в репозитории для материалов исследования.

## Зачем он нужен

Без подключения проекта легко получить пустые узлы с внутренними
идентификаторами в видимом тексте, смесь языков или шаблонный тон
«как у модели». Скилл держит разговор на естественном языке, сверяет
формулировки с «холодным читателем» и перед записью прогоняет тексты через
«Тексты на понятном языке» — в том числе против признаков шаблонного письма
языковых моделей из Wikipedia.

## Как пользоваться

В установленном ResearchOS попросите ассистента подключить репозиторий
или «подключить проект». Агент следует файлу `SKILL.md` в
`agents/skills/koi-project-onboard/` (есть зеркало в `.cursor/skills/`).

## Состав пакета

- `SKILL.md` — сценарий для агента
- `framing-checks.md` — проверки формулировок
- `onboard-brief.template.md` — шаблон краткого описания целей
- `manifest.yaml` — карточка для каталога Hub
